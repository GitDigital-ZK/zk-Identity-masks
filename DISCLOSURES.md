Responsible Disclosure Guidelines

We are committed to working with the security community to verify, reproduce, and respond to legitimate security vulnerabilities. If you believe you have found a security vulnerability in zk-Identity-masks, please follow the process outlined in SECURITY.md.

Our Commitment

· We will acknowledge receipt of your report within 2 business days.
· We will provide an estimated time frame for addressing the vulnerability.
· We will notify you when the vulnerability is fixed and credit you (if desired) in the release notes.

Scope

Only the following components are considered in scope:

· Core cryptographic implementations (Zig, Mojo, TypeScript)
· Smart contract / registry logic (if applicable)
· SDK APIs handling private data

Out of scope: documentation examples, test suites, and third-party dependencies (report those to the respective projects).

Recognition

We maintain a Hall of Fame for security researchers who help us improve the security of zk-Identity-masks. If you would like to be credited, please let us know when you submit your report.
