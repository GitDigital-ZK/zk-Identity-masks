Zig Modules for zk-Identity-masks

This directory contains the Zig implementation of core components for zk-Identity-masks.

Modules

· mask_verifier.zig – Verifies zero-knowledge proofs for identity masks.
· mask_registry.zig – Client for interacting with the mask registry (on-chain).
· hash.zig – Cryptographic hash functions (Poseidon) used in commitments.
· serialization.zig – Serialization/deserialization of proofs and public inputs.

Usage

Import the modules in your Zig application:

```zig
const mask_verifier = @import("mask_verifier.zig");
const registry = @import("mask_registry.zig");

pub fn main() !void {
    var verifier = mask_verifier.MaskVerifier.init(allocator, vk);
    defer verifier.deinit();
    // ...
}
```

Building

To build the Zig modules, ensure you have Zig 0.11 or later. Run:

```bash
zig build test
```

to run the tests.

Integration with Mojo/TypeScript

The Zig verifier is designed to be compatible with proofs generated by the Mojo prover. Both use the same circuit definitions and hash functions.
